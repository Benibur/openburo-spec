# Milestone v1.0 — Audit Report

**Status:** ✓ PASSED — milestone is ready to ship.

## Summary

Milestone v1.0 delivers the OpenBuro Server reference implementation in five sequential phases:

1. **Phase 1 — Foundation** (shipped 2026-04-10): Go 1.26 project skeleton, `config.yaml` loader, `slog` logger injection, `GET /health`, CI with `go test -race` + `vet` + `staticcheck`, Makefile. 7 requirements.
2. **Phase 2 — Registry Core** (shipped 2026-04-10): `internal/registry` pure domain package — `Manifest` + `Validate`, atomic JSON persistence with rollback, symmetric 3×3 MIME matching, `Store.Capabilities(filter)` view with deterministic sort. 20 requirements.
3. **Phase 3 — WebSocket Hub** (shipped 2026-04-10): `internal/wshub` byte-oriented pub/sub on `coder/websocket v1.8.14` — non-blocking fan-out, drop-slow-consumer, leak-free `Subscribe` with `CloseRead + defer removeSubscriber`, 1000-cycle goroutine-leak test. 5 requirements.
4. **Phase 4 — HTTP API** (shipped 2026-04-10): `internal/httpapi` transport layer — full REST + WebSocket contract, timing-safe Basic Auth, mutation-then-broadcast ordering, snapshot-on-connect, `rs/cors` with shared allow-list, 70 tests including REST/WS round-trip. 26 requirements.
5. **Phase 5 — Wiring, Shutdown & Polish** (shipped 2026-04-10): `cmd/server/main.go` compose-root (99 lines), two-phase graceful shutdown (PITFALLS #6), optional TLS, `README.md`, `TestGracefulShutdown`. 5 requirements (+ 1 test gate).

Total: **64 requirements across 5 phases. 64/64 complete. Zero orphans. Zero deferred.**

## Architectural Invariants Held

All load-bearing invariants verified at audit time:

1. **Unidirectional dependency graph** — `go list -deps ./internal/registry | grep -E 'wshub|httpapi'` is empty. `go list -deps ./internal/wshub | grep -E 'registry|httpapi'` is empty. The `httpapi` package is the ONLY place where `registry` and `wshub` meet. The ABBA deadlock is structurally impossible.
2. **No global logger** — `grep -rE 'log/slog|slog\.Default' internal/*/*.go cmd/server/*.go | grep -v _test.go` is empty. Every component receives its `*slog.Logger` through its constructor.
3. **No `time.Sleep` in tests** — `grep -n 'time\.Sleep' internal/*/*_test.go cmd/server/*_test.go` is empty. All timing assertions use `require.Eventually` (PITFALLS #16).
4. **No `InsecureSkipVerify`** — `grep -rn 'InsecureSkipVerify' internal/httpapi/*.go | grep -v _test.go` is empty (PITFALLS #7).
5. **`internal/config` is not imported by `internal/httpapi`** — `grep -rn '"github.com/openburo/openburo-server/internal/config"' internal/httpapi/*.go` is empty. The compose-root translates `config.Config` into `httpapi.Config`.
6. **No credential material in logs** — `TestAuth_NoCredentialsInLogs` captures slog output across full middleware chain and asserts 8 forbidden substrings (Authorization header, bcrypt prefix, plaintext username/password, Base64 encoding).
7. **Timing-safe Basic Auth** — `bcrypt.CompareHashAndPassword` runs on every request via `dummyHash` fallback; final authorization uses `subtle.ConstantTimeCompare` on a byte tuple. 13 grep matches on the load-bearing identifiers in `auth.go`.
8. **Mutation-then-broadcast ordering** — in both `handleRegistryUpsert` and `handleRegistryDelete`, `store.Upsert/Delete` line number is strictly less than `hub.Publish` line number (verified statically in plan 04-03 acceptance criteria).
9. **Two-phase shutdown** — in `cmd/server/main.go`, `httpSrv.Shutdown` (line 86) runs before `hub.Close()` (line 88). Proven end-to-end by `TestGracefulShutdown` which binds a real listener, cancels the context, and asserts `run()` returns nil within 20 s.

## Deviations Ledger

Over the course of the milestone, **no Rule 2 (user-contract) deviations occurred** — every user decision captured in CONTEXT.md files was honored. The deviations that did occur were all Rule 1 (latent bugs caught during execution) or Rule 3 (non-blocking plan refinements). Four of them fall into a recurring pattern worth noting:

- **Grep-gate self-references** — 4 plans landed where a grep gate in the acceptance criteria tripped on the plan's own documentation text (e.g. `! grep slog.Default` tripping on "no slog.Default" in a doc comment). Fixed each time by rewording the doc comment.
- **`-race` inflates bcrypt cost** — verification timeouts that assumed non-race timing were too tight when running under `-race` (bcrypt cost 12 goes from ~250 ms to ~2 s). Fixed by raising timeouts.
- **`statusCapturingWriter` Hijacker forwarding** — latent bug from Phase 4 plan 04-01 where the middleware wrapper didn't forward `http.Hijacker`, breaking all WS upgrades. Caught and fixed in plan 04-04.
- **`json omitempty` on empty slices** — plan 04-04 assumed `omitempty` on a `Capabilities []CapabilityView` field would keep it in the output for snapshot events, but Go drops empty slices. Fixed with a dedicated `snapshotPayload` type.

These are all documented in their respective plan SUMMARYs. None required user input; all were auto-fixed by executors.
